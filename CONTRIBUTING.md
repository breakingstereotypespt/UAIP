# Contributing to UAIP

Thanks for helping improve UAIP.

## Good contributions
Helpful contributions include:
- clearer wording
- stronger examples
- benchmark cases
- provider integrations
- diagrams and teaching materials
- empirical evaluation results
- translations

## Pull request checklist
Before submitting:
- explain what changed
- explain why it helps
- update docs if protocol wording changed
- keep terminology consistent
- avoid overstating safety claims

## Project principles
UAIP should remain:
- portable
- human-readable
- model-readable
- ethically grounded
- lightweight
- vendor-agnostic

## Versioning guide
- wording-only patch: `v1.0.x`
- backward-compatible improvement: `v1.1`
- breaking conceptual change: `v2.0`

## Community standard
Be evidence-minded, respectful, and transparent about uncertainty.
