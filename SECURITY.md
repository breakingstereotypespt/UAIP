# Security Policy

## Scope
This repository contains prompts, specifications, examples, and evaluation scaffolding. It is not a security product.

## Reporting concerns
If you discover:
- prompt injection weaknesses in examples
- unsafe benchmark design
- misleading claims in documentation
- harmful misuse pathways created by this repo

please open a private report with the maintainers before public disclosure if possible.

## What to report
Examples:
- a benchmark that incentivizes unsafe behavior
- an integration snippet that encourages deceptive or harmful use
- documentation that overstates safety guarantees
- example code that leaks secrets or mishandles environment variables

## Disclosure philosophy
We prefer:
- responsible disclosure
- transparent fixes
- documented changes in the changelog

## Limitations
UAIP does not guarantee safe output. It is a protocol pattern that should be tested in context.
